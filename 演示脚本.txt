========================================
作业3演示视频脚本（3-5分钟）- Web界面版
========================================

【第一部分：系统启动和配置】（1分钟）
--------------------------------------------------
1. 打开终端/命令行
2. 展示项目目录结构（快速）
   - 说明：这是基于LangGraph的多角色协作智能体系统
   - 展示主要文件夹：agents/, workflow/, tools/
3. 展示配置文件（快速）
   - 打开 docker-compose.gpu.yml
   - 说明：使用Docker部署，支持GPU，端口8000映射
4. 启动系统
   - 命令：docker-compose -f docker-compose.gpu.yml up -d
   - 等待容器启动（约10秒）
   - 检查状态：docker-compose -f docker-compose.gpu.yml ps
5. 启动Web服务（主服务，端口8000）
   - 命令：docker exec -d homework3-langgraph-agents-gpu python app.py
   - 说明：主服务，包含完整工作流
6. 启动三个智能体独立服务
   - 命令：
     docker exec -d homework3-langgraph-agents-gpu python receptionist_app.py
     docker exec -d homework3-langgraph-agents-gpu python analyst_app.py
     docker exec -d homework3-langgraph-agents-gpu python solution_expert_app.py
   - 说明：三个智能体分别在8001、8002、8003端口
7. 说明：系统已成功启动，包含主服务（8000）和三个智能体独立服务（8001-8003），以及4个MCP服务

【第二部分：LangGraph工作流智能路由演示】（1分钟）
--------------------------------------------------
说明：主服务（8000端口）使用完整的LangGraph工作流，实现智能路由

1. 打开主服务Web界面
   - 访问：http://localhost:8000
   - 说明：这是主服务，包含完整的LangGraph工作流

2. 演示智能路由（30秒）
   - 输入："你好，我想咨询一下"
   - 展示：系统自动路由到接待员 → 问题分析师 → 解决方案专家
   - 说明：LangGraph根据问题类型自动决定路由路径

3. 演示条件路由（30秒）
   - 输入："我的订单12345有问题"
   - 展示：接待员识别 → 分析师分析 → 调用工具 → 专家提供方案
   - 说明：根据问题复杂度，系统自动决定是否需要调用工具

【第三部分：分别测试三个智能体角色】（1分钟）
--------------------------------------------------
说明：每个智能体都有独立的服务端口，可以单独测试每个角色的功能

1. 启动三个智能体独立服务（在容器内）
   - 后台运行：
     docker exec -d homework3-langgraph-agents-gpu python receptionist_app.py
     docker exec -d homework3-langgraph-agents-gpu python analyst_app.py
     docker exec -d homework3-langgraph-agents-gpu python solution_expert_app.py

2. 测试接待员智能体（20秒）
   - 访问：http://localhost:8001
   - 展示：接待员服务页面
   - 使用API测试："你好，我想咨询一下"
   - 说明：单独测试接待员的功能

3. 测试问题分析师智能体（20秒）
   - 访问：http://localhost:8002
   - 展示：分析师服务页面
   - 使用API测试："我的订单12345有问题"
   - 说明：单独测试分析师的功能

4. 测试解决方案专家智能体（20秒）
   - 访问：http://localhost:8003
   - 展示：解决方案专家服务页面
   - 使用API测试："帮我解决订单查询的问题"
   - 说明：单独测试解决方案专家的功能

【第四部分：Web界面演示 - 4个MCP服务】（2分钟）
--------------------------------------------------
说明：通过Web界面提问来演示各个MCP服务，展示工具集成功能

1. 时间MCP服务演示（30秒）
   - 在Web界面输入："现在几点了？" 或 "今天几号？"
   - 展示：返回时间/日期信息
   - 说明：使用了时间MCP服务（Python MCP服务器），展示工具调用能力

2. 12306火车票查询MCP演示（40秒）
   - 在Web界面输入："我想查询北京到上海的火车票"
   - 展示：系统处理过程（三个智能体协作 + 工具调用）
   - 展示：返回真实的车次数据（多个车次信息）
   - 说明：使用了12306 MCP服务（npx 12306-mcp），返回真实数据

3. 知识库/记忆系统MCP演示（30秒）
   - 在Web界面输入："常见问题有哪些？" 或 "如何查询订单？"
   - 展示：返回知识库搜索结果
   - 说明：使用了Memory MCP服务（@modelcontextprotocol/server-memory）

4. 文件系统MCP演示（20秒）
   - 在Web界面输入："查看系统日志" 或 "列出数据目录"
   - 展示：返回文件系统操作结果
   - 说明：使用了Filesystem MCP服务（@modelcontextprotocol/server-filesystem）

【第五部分：总结和结束】（30秒）
--------------------------------------------------
1. 展示Web界面会话历史
   - 说明：可以看到不同智能体的回复
   - 展示：三个智能体分别处理不同类型的问题
2. 总结系统特点
   - LangGraph工作流实现智能路由（主服务8000端口）
   - 三个智能体各司其职（接待员、分析师、专家）
   - 三个智能体独立服务（8001-8003端口）可单独测试
   - 4个真实MCP服务集成（时间、12306、知识库、文件系统）
   - Web界面友好交互
   - 支持多轮对话
3. 说明：系统已完整实现作业要求的所有功能
